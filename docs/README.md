# Documentation

This folder contains project documentation and planning state.

## Structure

```
docs/
├── README.md           # This file
└── plans/              # Planning folders (persistent state)
    ├── live/           # Active work in progress
    └── completed/      # Finished plans (reference)
```

## Plans Folder

Planning folders track work across Claude Code sessions using YAML-based state management.

### Live Plans (`plans/live/`)

Contains active work in progress. Each subfolder follows the naming convention `YYMMDDXX_Description` (e.g., `260104AE_agenticguidance`), where `XX` is the Worktree Identifier (e.g., `AE` for AgenticEngineering).

**Note:** This folder may be empty when no active work exists. Plans are created here when work begins and moved to `completed/` when finished.

Example folder structure (when active work exists):
```
plans/live/
└── YYMMDDXX_Description/   # e.g., 260104AE_agenticguidance (XX=AE triggers worktree association)
    ├── plan_*.yml        # Active plan files
    ├── orchestration_*.mmd  # MMD execution graph (generated by orchestration-planning)
    └── completed/        # Finished tasks within this plan
```

Additional subfolders may include:
- `analysis/` - Friction analysis and notes
- `audit/` - Audit reports

## Plan File Format

Plan files use YAML with this structure:

```yaml
name: plan-name-kebab-case
status: pending  # pending | in_progress | completed | removed
priority: high   # high | medium | low
created: "YYYY-MM-DD"
phase_type: build  # build | test | teach | deploy

objective: |
  What this plan accomplishes and why.

context: |
  Background information and constraints.

phases:
  - name: "Phase Name"
    status: pending
    description: "Phase description"
    tasks:
      - id: "task_001"
        name: "Task name"
        status: "pending"  # pending | in_progress | completed
        description: "Task description"
```
